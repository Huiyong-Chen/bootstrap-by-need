# 001. 只有浏览器听得懂的语言

## 当前阶段任务

实现一个最基础的“随机题库”：页面上显示题目，点击按钮切换下一题。

---

## 架构师的内心独白

当我创建这个文件夹时，我的肌肉记忆差点让我打出了 `npm init`。

但我停住了。

我问自己：**“为了在屏幕上显示一行字，我真的需要一个 `package.json` 吗？”**

如果不安装任何工具：

- 我不需要等待 `node_modules` 下载。
- 我不需要配置 Webpack 或 Vite 的入口。
- 我不需要担心 TypeScript 的类型报错。

我只需要新建文件，写代码，然后双击 `index.html`。这种“即刻满足感”是工程化给不了的。

### 关于目录结构的纠结

一开始我想把 CSS 和 JS 直接写在 `index.html` 的 `<style>` 和 `<script>` 标签里。
但即使是 MVP（最小可行性产品），我也无法忍受超过 50 行的 HTML 文件。
所以，我退让了一步，建立了 `js/` 和 `styles/` 目录。
这是一种**极其廉价的组织方式**——没有构建成本，只有文件管理成本。

### 关于数据的处理

数据从哪里来？
去请求一个 JSON 文件？不。
如果在本地直接用 `fetch('data.json')`，浏览器会因为 `file://` 协议报 CORS 错误。
为了解决这个错误，我就得引入 `http-server` 或者 `five-server`。
**看，复杂度就是这样悄悄溜进来的。**
为了这一个功能，引入一个服务器？不划算。

所以，我决定把数据做成一个常量数组，直接写死在 `js/index.js` 里。
虽然后期肯定要改，但现在，它是最快能让我看到东西跑起来的方法。

**现在的目标是：Keep It Stupid Simple.**
